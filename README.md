# Our Medicals - B2B Medical Wholesale Platform

A premium, ultra-minimal monorepo template for a medical wholesale e-commerce platform.

## Architecture

* **Frontend:** React + Tailwind CSS v4 + Vite + Clerk (Identity) + Zustand (Persistent Cart Store)
* **Backend:** Python + FastAPI + Uvicorn + PyJWT (Auth Verify) + Razorpay SDK (Payments & Refunds)
* **Database:** Supabase (PostgreSQL)
* **Version Control:** Git (Local & remote tracking)

---

## Getting Started

### 1. Database Setup (Supabase)

Create a free account on [Supabase](https://supabase.com). Create a new project and run the following SQL statements in the **SQL Editor** to initialize the database tables:

```sql
-- 1. Create the Salts table
CREATE TABLE salts (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name TEXT UNIQUE NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL
);

-- 2. Create the Medicines table
CREATE TABLE medicines (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    salt_id BIGINT REFERENCES salts(id) ON DELETE CASCADE NOT NULL,
    name TEXT UNIQUE NOT NULL,
    url TEXT,
    image_url TEXT,
    composition TEXT,
    formulation TEXT,
    status TEXT,
    description TEXT,
    side_effects TEXT,
    dosage TEXT,
    price NUMERIC(10, 2) NOT NULL DEFAULT 9.99,
    stock INTEGER NOT NULL DEFAULT 100,
    min_order_quantity INTEGER NOT NULL DEFAULT 10,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL
);

-- 3. Create indexes for quick search
CREATE INDEX idx_medicines_name ON medicines(name);
CREATE INDEX idx_salts_name ON salts(name);
```

### 2. Backend Setup

1. Navigate to the backend directory:
   ```bash
   cd backend
   ```

2. Create a virtual environment and activate it:
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   ```

3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

4. Create a `.env` file in the `backend/` folder:
   ```env
   # Database keys (from Supabase Project Settings -> API)
   SUPABASE_URL=https://your-project-id.supabase.co
   SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

   # Clerk JWT Authentication (optional for local testing, required for production verification)
   # Issuer URL looks like: https://clerk.yourdomain.com/.well-known/jwks.json
   CLERK_JWKS_URL=
   ```

5. Seed the database with `medicines.json` (make sure your Supabase keys are in `.env` first!):
   ```bash
   python seed.py
   ```

6. Start the local API server:
   ```bash
   uvicorn main:app --reload
   ```
   The API will be available at `http://localhost:8000`. You can test endpoints and view the auto-generated Swagger UI at `http://localhost:8000/docs`.

---

### 3. Frontend Setup

1. Navigate to the frontend directory:
   ```bash
   cd frontend
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Create a `.env` file in the `frontend/` folder:
   ```env
   # Clerk Publishable Key (from Clerk Dashboard -> API Keys)
   VITE_CLERK_PUBLISHABLE_KEY=your-clerk-publishable-key
   ```

4. Run the local development server:
   ```bash
   npm run dev
   ```
   The website will run at `http://localhost:5173`.
